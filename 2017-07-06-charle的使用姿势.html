charle的使用姿势

发表于 2017-07-06 | 分类于 工具 | 阅读次数

HTTP抓包

http抓包比较简单，基本只要打开charles，然后打开浏览器，上个网页，就可以看到charles抓到的信息。

抓取chrome浏览器的请求

  1. 打开charles(默认端口:8888)
  2. 打开chrome浏览器，然后打开chrome上的代理工具,输入127.0.0.1:8888 将chrome的请求代理到本地的charles上。再发出请求就能被抓取到了。

抓取移动端的请求

  1. 打开charles(默认端口:8888
  2. 记下本机电脑连接的ip
  3. 打开手机，连上wifi，选择http代理，填写好之前记下的电脑ip地址和端口号。现在该手机发送的请求就能被抓取到了。

https抓包

  1. 安装charles的根证书
    1. 个人电脑
      • HELP =》 SSL Proxying =》 Install Charles Root Certificate
    2. 移动端
      • 打开Charles，在Menu选择Help > SSL Proxying > Install Charles Root Certificate on a Mobile Device or Remote Browser，弹出提示框；按照提示配置好移动端ip地址
      • 浏览器输入网址 http://charlesproxy.com/getssl =》 安装证书
      • 安卓手机上需要设置密码锁后才能安装证书
  1. 将证书添加信任
  2. 在charles 选择一个https请求，右击弹出菜单，选择 “Enable SSL Proxying” or 使用通配符 *:443
  3. 完成

提升开发效率

  1. 快速定位 BUG
  2. 快速模拟 请求中，请求失败的情况
  3. 映射本地文件
  4. 模拟不同网络环境下的页面情况

官网文档部分翻译

Using Charles

  1. Sessions(会话)
    • 你的会话窗口包含了所有的记录信息。当你打开花瓶的时候，就会有一个默认的会话窗口自动创建。、
    • 正常情况下，你只需要一个会话窗口。除非你感到需要使用多个会话。当会话窗口被信息填满的时候，你可以清空它。
    • 会话能够被保存以及再次打开。没准你的同事会用到~。
  2. Recording(记录)
    • 记录是花瓶最重要的功能。响应和请求是会记录到当前会话窗口的，以供我们稍后的检查和分析。
    • 记录哪些请求取决与花瓶的配置。通常会记录http和https请求来自于当前电脑的浏览器以及该机上的一些其他应用。
    • 如果把记录关闭，花瓶仍然会像之前一样通过所有请求，只是它们不会被记录在会话中。
  3. Requests & Responses(请求和响应)
    • 一个请求及其响应是记录在花瓶中的基本元素。
    • HTTP/HTTPS由一对对的请求响应组成。

Proxying

  1. Throttling(节流)

    • 花瓶能够调节你的互联网带宽和延迟。这使你用高速宽带来模拟调制解调器。
    • 带宽能够控制任意字节每秒，可以用来模拟任何连接速度。
    • 延迟也可以设置为任意数量的毫秒数。延迟模拟能够实现更慢的连接，这个延迟是发生在请求和另一个接手请求之间。

  2. Breakpoints(断点)

    • 断点工具让你能够截断请求和响应在他们经过花瓶。你能够检测和编辑请求和响应，然后决定是否允许进行和阻止请求。
    • 每个断点匹配一个通过位置匹配模式的URL。每个断点也能单独加给请求或者响应，也可以同时加在两者身上。

  3. SSL Proxying(SSL 代理)

    • SSL Proxying
      • 花瓶作为HTTPS的中间代理，让你能够查看在web浏览器和SSL的web服务器之间的报文。
      • 花瓶能够实现SSL代理通过成为一个中间人的角色。代替你的浏览器查看服务器的证书。花瓶动态生成一个证书提供给服务器，而且出示自己的根证书。花瓶接受服务器证书，知道你的浏览器接受花瓶证书。因此你会看到一个安全警告，像是根证书不受信任。如果你添加了花瓶的CA证书到信任区，你将不会看到任何警告。
      • 花瓶仍然通过SSL连接到web服务器。这个SSL通讯流程是从web浏览器到花瓶，再从花瓶到web服务器。
      • 这是一个在我们调试SSL应用时最基本的功能。
      • 我们能够在Proxy面板上打开或者关闭SSL代理。如果SSL代理被关闭了，花瓶只是将SSL流量直接导向目标WEB服务器。
    • Choosing hosts to SSL Proxy
      • 首先你得确定你要代理的主机能够进行SSL代理。然后在SSL Proxy Settings选项中，填写你的主机名。或者，你也能够在结构视图中右击来打开SSL代理。
      • 如果你想SSL代理所有主机名可以在SSL Proxy Settings中填写到host列表。
        -*Trusting Charles’s SSL Certificates

  4. Reverse Proxy（反向代理）

    • 反向代理能够在本地端口创建一个web服务器，用来代理请求到远程服务器。所有反向代理的请求和响应可以被记录在花瓶中。
    • 如果你有一个客户端应用程序不支持使用HTTP代理，或者你想避免配置使用代理，那么你可以使用反向代理。创建一个反向代理到原始目标服务器然后连接到本地端口的客户端;反向代理对于本地客户端应用是透明的，它能够查看到之前你无法查看到的流量。
    • HTTP or HTTPS/SSL
      • 花瓶能够创建一个反向代理作为一个HTTP和HTTPS服务器。对于HTTP可能是80端口，而HTTPS可能是443接口。

  5. Port Forwarding(端口转发)

    • 使用端口转发工具可以把任意的TCP/IP或者UDP接口从花瓶转发到远程主机。这样就可以在花瓶调试任意的协议。

Tools

  1. No Caching Tool(禁止缓存)

    • 禁止缓存工具能够阻止浏览器从缓存加载资源。所有的资源会从远端站点加载，你看到的内容总是最新版本的。
    • Selected Locations
      • 该工具能够针对所有请求或者针对部分请求。当使用”Only for selected locations”选项时，你能够限制指定的主机或者使用简单但是功能强大的匹配模式
    • About caching
      • 缓存可以使客户端应用程序，像浏览器，使用本地副本的响应，而不是再次从网站上请求。缓存是web中非常重要的组成部分。经常会出现在多个页面上使用相同的资源，如果重新加载每个页面的每个资源会大大降低浏览体验。但是缓存意味这你见到的不会是这个资源的最新版本，在你开发你的web应用程序的时候这是个问题，因为你总是想要看到最新的变化。
    • How it works
      • 禁止缓存工具通过控制HTTP响应缓存标头来阻止缓存操作。”If-Modified-Since and If-None-Match”部分会从请求中被移除。”, Pragma: no-cache and Cache-control: no-cache “部分会被添加。”The Expires, Last-Modified and ETag”部分从响应中移除，”Expires: 0 and Cache-Control: no-cache”部分会被添加

  2. Block Cookies Tool

    • Block Cookies Tool能够阻止发送和接受cookies。它能够测试你的网站，如果你禁用了cookies。注意：web爬虫经常不会支持cookies，所以这个工具也可以模拟爬虫对网站搜索规则。
    • Selected Locations
      • 同 No Caching Tool
    • How it works
      • 移除了请求中的cookies部分，阻止cookie值从本地客户端发送到远程服务器。移除了响应中的Set-Cookie部分，用来阻止接受远程服务器发送到本地客户端的cookies。
  3. Map Remote Tool（远程映射）
    • 该功能能够改变请求位置,通过映射配置，使响应发送到新的位置，就好像它原本的请求。
    • 该功能可以提供全部或者部分的映射从一个站点到另一个站点。例如，你能够提供一个网站的子目录xxx.com/charles/ 到 localhost/charlesdev/ 或者提供所有的目录文件。通过通配符加上后缀。
    • Usage Suggestions
      • 如果你有一个开发中的网站，想要将开发中网站的一些请求发送到线上网站进行测试，那么远程映射是非常有用的。例如，你想从你的开发服务器中提供css和图片，远程映射就像这样 live.com/css to dev.com/css 或者 live.com/*.css to dev.com.
    • Mapping Types
      • 映射一个目录到另一个目录，例、 xk72.com/charles/ to localhost/charlesdev/
      • 映射一个文件夹到另一个文件夹，例、xk72.com/charles/download.php to abc.com/testing/test.html
      • 映射目录中的部分文件到目录，例、xk72.com/charles/*.php to localhost/charlesdev/
      • 如果你不在目标映射中指定一个路径，那么该网址的路径部分将不会被更改。如果你想映射到根目录，在目标路径字段放一个/。
    • HTTPS
      • 远程映射工具能够映射一个HTTP请求到一个HTTPS目的，反之亦然，所以你可以有HTTP或HTTPS映射到他们的对立面。
  4. Map Local Tool（本地映射）
    • 本地映射能够让你使用本地文件作为远程网站的一部分。你可以将你的本地文件模拟在线上的效果。本地文件的内容会返回到客户端，就好像远程响应的内容一样。
    • 如果你不得不上传一个文件到网站用来测试，那么本地映射可以大大加快开发和测试速度。而且用这种方法可以在测试环境安全的测试。
    • Dynamic files
      • 动态文件，像包含了服务端脚本的文件，不能被本地映射所执行。所以如果文件中有动态脚本，脚本将返回给浏览器，这可不是我们想要的结果。
      • 如果你想使用动态文件，可以去了解了解远程映射工具。
    • How it works
      • 当请求与本地映射匹配时，它会检查与路径匹配的本地文件。他不会匹配查询字符串。如果在本地发现请求文件然后它就会被作为响应返回，就好像从远程服务器下载的文件。如果请求文件没有在本地发现，那么就会像平时一样从远程服务器下载。
    • Examples
      • 如果你测试你的css、swf或者图片你可以映射这些文件类型到本地。所以你可以用这些开发中的文件浏览线上网站。
    • Location Matching
      • 每个位置匹配可以包含协议，主机，端口和路径，用来匹配特定的URLs。可以使用通配符来定位。
